# Hirono (小野) 3D 建模与特效集成设计方案

## 1. 需求锁定 (Understanding Lock)
* **目标**：在 C++ 中为 Hirono (小野) 开发全新的 3D 模型，替换原本在格斗游戏和模型查看器中使用的 2D 纸片层级板羽。
* **建模规格**：手工 3D 几何建模（不使用外部 OBJ 加载），包含 11~13 个金黄色发梢花瓣发块、深绿色大衣身躯、黄色围巾、半透明红色披风以及双手捧持的半透明玫瑰玻璃罩（内含红色玫瑰和绿色花茎）。
* **非目标**：不制作月球底座，保持人物及其手中玻璃罩道具在原点上方的相对位置。
* **性能要求**：使用 OpenGL 显示列表（Display Lists）缓存机制缓存发梢花瓣、玻璃罩等不形变几何体，配合分层关节坐标矩阵变换（`glPushMatrix`/`glPopMatrix`），确保同屏稳定 60 FPS。

## 2. 决策日志 (Decision Log)
| 决策项 | 备选方案 | 选用理由 |
| :--- | :--- | :--- |
| **层级建模 + 显示列表缓存** | 纯过程式每帧计算渲染 | 选用前者。极大地减少 CPU-GPU 数据提交开销，保障 60 FPS 满帧，并且完美对齐已有的 Dimoo 建模架构。 |
| **排除月球底座** | 绘制带陨石坑底座 | 选用前者。用户明确提出不需要底座，以聚焦角色本体。 |
| **3D 玫瑰玻璃罩建模** | 简化玫瑰罩或画贴图 | 选用前者。利用半透明混合（`alpha = 0.35`）和深度写入控制绘制 3D 玻璃体、花茎和红色玫瑰花，提供卓越的高端质感。 |
| **花瓣普攻粒子系统** | 简易粒子或画点 | 选用前者。设计 3 片具有正旋轨迹和自转（spin）的 2D 红色四边形花瓣粒子，命中后爆裂，提升视觉真实感。 |

## 3. 技术设计方案 (Technical Design)

### 3.1 几何体构造树
```
[Hirono Root Transform]
  ├── [Opaque Pass]
  │     ├── 躯干 (Ellipsoid, 深绿色)
  │     ├── 头部 (Ellipsoid, 肤色 + hirono_face.jpg 贴图)
  │     │     ├── 11~13发梢发块 (Ellipsoid 渐变插值, 柠檬黄->白)
  │     │     └── 脸颊雀斑 (GL_POINTS)
  │     ├── 双臂 (Capsules, 绿袖) -> 手掌 (Ellipsoid)
  │     └── 黄色围巾段 (Quads, 黄色, 正弦微风摆动)
  │
  ├── [Translucent Pass (glDepthMask(GL_FALSE))]
  │     ├── 红色披风 (Quads, 半透明红色, 惯性滞后偏摆)
  │     └── 玫瑰玻璃罩 (Cylinder + Hemisphere, 透明蓝, alpha=0.35)
  │           ├── 玫瑰木底座 (Cylinder, 棕色, 刻字 "B-612")
  │           ├── 玫瑰花茎 (Cylinder, 绿色)
  │           └── 玫瑰花瓣 (Quads, 红色, 围绕排列)
  └── [Particle Pass]
        ├── 玫瑰花瓣粒子 (Quads)
        └── B-612 扩展金圈 (GL_LINE_LOOP)
```

### 3.2 动态变换与动作融合
1. **呼吸 (Idle)**：整体沿 Y 轴进行 $breathScale = 1.0 + 0.01 \sin(1.57 t)$ 极其微小的缩放，围巾和披风末端随时间微幅振荡。
2. **移动 (Walk)**：身体绕 Z 轴作 $\pm 3.0^\circ$ 的小碎步倾晃，头部 Y 轴做 $\pm 0.015$ 的上下颠簸。
3. **披风惯性延迟**：披风偏摆角以身体平移偏移量为基础，附带 $0.25$ 秒的时间差相位延迟，产生自然的织物重力拉扯感。
